## 2. 简历列表页

**在哪里看到简历？**

- 路径：`/web/chat/recommend`，左侧边栏导航点击「**推荐**」进入推荐候选人列表
- Boss直聘没有独立的"收到的简历"列表。候选人来源有两类：

| 入口 | 说明 |
|------|------|
| **推荐牛人 Tab** | 系统根据招聘需求主动推荐的候选人，以卡片流形式展示 |
| **沟通 Tab** | 已发起过对话的候选人列表 |

本次调研主要针对**推荐牛人 Tab**。

**列表页能看到哪些字段？**

每张候选人卡片（`.geek-card-small`）展示以下信息：

| 字段 | 说明 |
|------|------|
| 姓名 | 脱敏显示（如 "杨**"） |
| 头像 | 圆形头像图 |
| 求职意向 | 期望职位名称 |
| 当前状态 | 在职 / 离职 / 应届 |
| 工作年限 | 如 "3年" |
| 学历 | 如 "本科" |
| 毕业院校 | |
| 期望薪资 | 如 "15-25K" |
| 活跃度标签 | 如 "今日活跃" |
| 技能标签 | 1–3 个标签 |
| 打招呼按钮 | 卡片右下角 |

**有没有分页？每页多少条？**

- **无分页**，采用**无限滚动（Infinite Scroll）**
- 初始加载约 **20 张**卡片，滚动到底部自动追加
- 已观测到 **115+ 张**候选人卡片可加载

**有没有筛选/搜索功能？**

- 有**筛选面板**，入口：列表顶部「**筛选**」按钮
- 筛选维度共 **6 个**：

| 筛选维度 | 示例选项 |
|----------|----------|
| 求职状态 | 离职-随时到岗 / 在职-考虑机会 / 在职-暂不考虑 |
| 工作年限 | 应届生 / 1年以内 / 1–3年 / 3–5年 / 5–10年 / 10年以上 |
| 学历要求 | 初中及以下 / 中专/中技 / 高中 / 大专 / 本科 / 硕士 / 博士 |
| 薪资范围 | 滑动区间选择 |
| 性别 | 男 / 女 |
| 年龄范围 | 输入区间 |

- 筛选器有**冷却倒计时**（通过 `/wapi/zpblock/recommend/filter/remain/time` 获取）
- **无关键词搜索框**，候选人发现仅依赖推荐算法 + 筛选器

**有没有"导出"按钮？**

- **没有直接导出按钮**
- 免费版不支持批量导出简历
- 简历获取需完整流程（见下节）

## 4. 打招呼 / 发消息

**从简历列表能不能直接打招呼？**

- **可以**

**打招呼的入口在哪？**

有**两个入口**，功能等同：

| 入口 | 位置 |
|------|------|
| 列表卡片入口 | 每张候选人卡片右下角的「**打招呼**」按钮 |
| 详情面板入口 | 详情面板顶部右侧的「**打招呼**」按钮 |

**打招呼消息有没有字数限制？**

- **≤ 150 字**
- 平台提供默认招呼语模板，可直接发送或修改后发送

**免费版每日配额**

| 操作类型 | 每日上限 |
|----------|----------|
| 主动发起沟通（打招呼） | **3 个**/天 |
| 主动查看简历 | **20 个**/天 |

配额耗尽后：点击「打招呼」按钮会直接弹出「**电话直拨**」付费功能推广弹窗，不会提示"今日配额已满"。

**打完招呼后，对话出现在哪里？**

路径：`/web/chat/index`，顶部导航「**沟通**」Tab

**沟通列表 Tab 分类**：

| Tab | 说明 |
|-----|------|
| 全部 | 所有沟通记录 |
| 新招呼 | 候选人主动发来的招呼（数字为未读数） |
| 沟通中 | 正在进行的对话（实时状态） |
| 已约面 | 已邀请面试的候选人 |
| 已获取简历 | 已获取附件简历的候选人 |
| 已交换电话 | 已互换联系方式的候选人 |
| 更多 ▾ | 展开更多分类 |
| + | 快捷发起新招呼入口 |

每条对话项展示：候选人头像 + 姓名 + 最新消息摘要 + 时间戳。

## 5. 完整简历获取流程

Boss直聘**不能直接下载简历**，必须完成以下完整流程：

```
推荐牛人（Tab）
    ↓
候选人卡片列表 → 点击候选人
    ↓
候选人详情页（右侧滑出详情面板，仅供查看，不可下载）
    ├─ 收藏 → 互动Tab → 收藏牛人
    ├─ 不合适 → 进入不合适候选人库
    ├─ 举报
    ├─ 转发牛人
    └─ 打招呼 → 自动发送预设招呼语
            ↓ 待候选人回复
沟通Tab（/web/chat/index）
    ↓ 候选人已回复（双向对话达成）
    ↓ 点击"求简历"按钮（仅在双向对话后可点击）
            ↓ 候选人同意发简历
    ↓ 简历发送过来
    ↓ 点开详情 → 可下载（下载到本地文件夹）
```

> 注意：整个流程依赖候选人主动配合，周期不确定，自动化难度较高。

## 8. 页面技术特征

**页面是 SPA（单页应用）还是传统多页？**

- **SPA（单页应用）**
- 前端框架：**Vue.js**（DOM 可见 `data-v-*` 属性，`__vue_app__` 挂载在根节点）
- 打包工具：Webpack（`/static/js/app.js`，版本号 `v6183`）
- 路由方式：`history.pushState`，URL 切换无整页刷新

**核心架构：三层 iframe 嵌套**

```
/web/chat/recommend（外层主页）
  └─ /web/frame/recommend/（候选人列表 iframe）
       └─ /web/frame/c-resume/（简历详情 iframe）
```

**Network 面板里能不能看到 API 请求？**

- ✅ **能**。共捕获 **112+ 条 /wapi/** 请求
- 全部为 **XHR / Fetch 类型**，状态码均为 **200**
- API 命名空间：`https://www.zhipin.com/wapi/`

**简历数据是通过什么接口加载的？**

| 接口 | 方法 | 说明 |
|------|------|------|
| `/wapi/zpblock/recommend/major/data` | GET | 推荐候选人列表（主数据） |
| `/wapi/zpblock/recommend/filters?jobId=xxx` | GET | 筛选条件选项 |
| `/wapi/zpblock/recommend/filter/remain/time` | GET | 筛选器冷却倒计时 |

其他关键接口：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/wapi/batch/requests` | POST | 批量 API 请求（页面加载时触发 6+ 次） |
| `/wapi/zprelation/friend/getBossFriendListV2.json` | POST | 沟通列表（已联系候选人） |
| `/wapi/zpitem/web/chat/message/list/box` | GET | 消息列表 |

## 9. 付费模式与配额（补充调研）

**笼中仙实际使用的方案：竞招职位 · 畅聊版**

| 权益项 | 畅聊版配额 |
|--------|-----------|
| 每日查看简历 | 不限 |
| 每日主动沟通（打招呼） | 50次 |
| 每日回聊（回复候选人主动消息） | 不限 |
| 有效期 | 30天/单岗位 |
| 价格区间 | 108-388元/单岗位/30天（因城市而异） |

**关键规则：**
- 付费按单个岗位绑定，不跨岗位、不跨账号通用
- 候选人主动发来的沟通，回复不消耗主动沟通次数
- 每日权益当日清零，不累计不结转
- VIP会员权益不覆盖竞招岗发布权限，需单独购买

**Boss直聘岗位分类：**

| 类型 | 说明 |
|------|------|
| 普通职位 | 免费发布，基础配额 |
| 竞招职位 | 供不应求岗位，必须付费才能发布（前台、服务员、收银员等基层岗常见） |
| 集招职位 | 制造/供应链/物流行业，需购买沟通钥匙 |

## 10. 详情面板 vs 附件简历对比（补充调研）

**结论：详情面板信息足够做AI匹配打分，不需要获取附件简历。**

| 信息字段 | 详情面板（打招呼前可看） | 附件简历PDF（需候选人发送） |
|---------|:---:|:---:|
| 姓名（完整） | ✅ | ✅ |
| 年龄/性别 | ✅ | ✅ |
| 手机号 | ❌ | ✅ |
| 求职状态 | ✅ | ✅ |
| 期望职位/薪资/城市 | ✅ | ✅ |
| 工作经历（公司+职位+时间+描述） | ✅ 完整 | ✅ 完整 |
| 教育经历 | ✅ | ✅ |
| 技能标签/关键词 | ✅ | ✅ |
| 资格证书 | ✅ | ✅ |
| 个人优势/自我评价 | ✅ | ✅ |
| 活跃度标签 | ✅ | ❌ |

**对C1方案的影响：** 简历获取只需从详情面板抓取结构化信息即可，无需走完"打招呼→候选人回复→求简历→下载PDF"的完整流程。手机号在AI匹配打分阶段无用。
