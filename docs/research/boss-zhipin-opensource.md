## 五、技术实现方式对比

| 实现方式 | 优点 | 缺点 | 代表项目 |
|----------|------|------|----------|
| **Tampermonkey 脚本** | 部署简单，无需服务端 | 依赖DOM结构，UI变则失效 | Jobs_helper, BOSScript |
| **Playwright / Selenium** | 可控制真实浏览器，功能全 | 被检测风险高，性能开销大 | boss-zhipin-automation, BossZP-Spider |
| **Chrome 扩展** | 可调用浏览器API | 上架审核严，分发困难 | zhaopin-boss-chrome |
| **RPA 模拟操作** | 模拟真人操作，难被检测 | 速度慢，稳定性差 | boss-hiring-v2 |
| **MCP Server** | 专为AI Agent设计，易集成 | 功能有限，依赖官方接口 | boss-zhipin-mcp |
| **直接 API 调用**（逆向） | 速度最快 | 违反 TOS，封号风险最高 | boss-cli |
